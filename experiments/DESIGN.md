# 4D 动态器官建模论文实验设计

本文档按顶会/顶刊论文的论证需求来设计实验，目标是把实验部分组织成一套对“动态器官 4D 建模方法”成立的标准证据链，胃部自由手超声只是这套框架的一个实例落点。

## 1. 论文实验要回答的核心问题

建议全文围绕以下 5 个研究问题组织。

1. **RQ1：本文方法是否能在动态器官 4D 建模中同时提升几何精度、时间一致性和拓扑稳定性？**
2. **RQ2：方法的有效性是否能跨不同器官运动模式、观测条件和采样策略保持稳定？**
3. **RQ3：方法的关键组成部分是否分别带来明确、可重复的增益？**
4. **RQ4：面对稀疏采样、噪声、遮挡、位姿误差和跨主体差异时，方法是否仍然鲁棒？**
5. **RQ5：在真实临床场景中，模型输出是否具有解剖合理性、时序连续性和潜在应用价值？**

这 5 个问题共同构成一篇动态器官 4D 建模论文的主线：整体效果、跨条件稳定性、机制解释、鲁棒性和真实可用性。

## 2. 实验总矩阵

### 实验 A：标准基准主结果

**目标**：回答 RQ1，建立论文最核心的定量结果。

**数据要求**：使用具备逐时相几何真值的可控动态器官基准。该基准不必局限于当前胃部数据，应至少覆盖主体差异、运动差异和成像差异三个维度。

**建议对比家族**：

1. 逐时相独立重建方法
2. 模板传播或非刚性配准方法
3. 共享表示的动态重建方法
4. 本文完整方法

**主指标**：

1. 平均 CD
2. 平均 HD95
3. 法向一致性
4. 周期闭环误差
5. 水密率或流形率

2. 图 1：代表性时相可视化

### 实验 B：跨条件泛化评测

**目标**：回答 RQ2，证明方法不是只在某一固定数据协议下有效。

**建议划分维度**：

1. 跨主体泛化
2. 跨采样密度泛化
3. 跨成像质量泛化
4. 跨运动模式泛化

这一组实验的关键不是再做一张静态表，而是展示性能在不同测试分布下的稳定性边界。

### 实验 C：组件递进与结构消融

**目标**：回答 RQ3，说明方法为什么有效。

**C1. 组件递进**：

1. 基础逐时相重建
2. 基础逐时相重建 + 相位标准化
3. 相位标准化 + 共享拓扑 4D 时空形变表示
4. 相位标准化 + 共享拓扑 4D 时空形变表示 + 质量感知观测建模
5. 完整方法

**C2. 结构消融**：

1. 仅共享模板
2. 仅局部残差
3. 无低秩全局运动分解
4. 无质量感知观测权重
5. 无周期闭环或高阶时间正则
6. 改变表示容量或基数

重点是解释建模结构，而不是只做若干删项实验。

### 实验 D：鲁棒性压力测试

**目标**：回答 RQ4，说明方法在真实困难条件下的可靠性。

**建议退化设置**：

1. 稀疏时相采样
2. 局部视角缺失
3. 观测噪声增加
4. 位姿或时间同步误差
5. 局部区域弱支撑

### 实验 E：真实数据验证

**目标**：回答 RQ5，证明方法在真实临床数据上的可解释性与可用性。

**建议展示内容**：

1. 完整周期或完整时序可视化
2. 观测证据与重建表面对齐展示
3. 高形变区域、低支撑区域和失败案例对比

**建议无 GT 指标**：

1. 周期闭环误差
2. 邻相位位移平滑度
3. 水密率 / 流形率
4. 体积或主轴曲线平滑性
5. 专家盲评或规则化人工质检评分

### 实验 F：器官特定动态事件分析

**目标**：为具体任务补充功能性或局部动态证据，但不作为整篇论文的主叙事中心。

对于胃部场景，可在这一节评估局部收缩传播区域、关键功能区或局部动态事件；对于其他器官，也可以替换成相应的任务特定区域运动指标。该实验的定位应是“器官特定补充分析”，而非“整篇论文的唯一亮点”。

## 3. 主文与补充材料的版面分配建议

为了符合顶会论文的叙事节奏，建议把实验拆成主文必放和补充扩展两层。

**主文必放**：

1. 标准基准主结果
2. 跨条件泛化结果
3. 组件递进或关键结构消融
4. 真实数据验证

**主文可选**：

1. 一张代表性的鲁棒性退化曲线
2. 一组器官特定动态事件分析

**补充材料建议放置**：

1. 完整结构消融
2. 全部鲁棒性扫描
3. 更多真实病例
4. 多随机种子复现
5. 失败案例库与误差模式总结

## 4. 评价指标重构建议

建议将指标组织成四层，而不是围绕单一局部现象来搭建。

### 全局几何指标

1. CD
2. HD95
3. 法向一致性
4. 体积误差或尺度误差

### 时间一致性指标

1. 邻相位位移平滑度
2. 二阶时间加速度能量
3. 周期闭环误差
4. 跨时相拓扑一致率

### 结构合理性指标

1. 水密率 / 流形率
2. 自交率或异常面片比例
3. 观测支撑一致性
4. 形态曲线稳定性

### 任务特定补充指标

1. 区域级局部误差
2. 关键事件相位误差
3. 功能区动态曲线误差

## 5. 统计分析要求

如果目标是顶会/顶刊，正式论文结果建议遵守以下统计规范：

1. 每组关键实验至少运行 3 次不同随机种子。
2. 主表统一报告 `mean ± std`。
3. 主方法与最强基线进行配对显著性检验。
4. 除平均结果外，同时报告失败案例或最差样本分析。
5. 对退化实验报告完整曲线，而不是只截取少数点。

## 6. 从论文标准反推的落地优先级

即便后续仍然使用当前仓库来实现，实验设计本身也应先按论文标准定下来。推荐优先级如下：

1. 先确定标准基准、真实数据和泛化测试三层数据协议。
2. 再确定对比方法家族，而不是先看当前仓库里有哪些旧版本。
3. 然后锁定主表、泛化表、鲁棒性图和真实数据图这四类核心论文资产。
4. 最后再把器官特定局部动态分析作为附加证据接入。

## 7. 实验归档建议

建议论文结果按证据链而非脚本来源归档：

1. `main_benchmark/`：标准基准主结果
2. `generalization/`：跨条件泛化测试
3. `ablation/`：组件递进与结构消融
4. `robustness/`：退化压力测试
5. `real_world/`：真实数据验证
6. `task_specific/`：器官特定动态事件分析

这样整理后，论文写作会围绕“方法是否普适成立”来展开，而不是围绕“某个脚本跑出了哪些结果”来组织。
